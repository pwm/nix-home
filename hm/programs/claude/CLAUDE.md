# Personal preferences (pwm)

These are my personal, cross-project preferences.
Project-specific facts go in `AGENTS.md` or auto-memory.

## Writing & documents
- use markdown
- avoid em dashes (—) unless necessary; prefer commas, parentheses, or separate sentences
- clarity over jargon
- simplicity over complexity
- start all documents with YAML front matter adding a title, a date and optionally a list of tags
- save all documents into the project root's .docs directory unless told otherwise

## Dates & files
- use ISO dates: `YYYY-MM-DD`
- prefer descriptive kebab-case filenames

## Git commits
- do NOT add the `Co-Authored-By: Claude …` trailer on commits
- always add a plain english exec summary to the top of each commit message
